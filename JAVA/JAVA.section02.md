## 06. 자바 프로그램 실행
**주의!!!!!!!!!!점**  
자바 언어는 대소문자 구분함  
대소문자가 다르면 오류가 발생할 수 있음  
들여쓰기 안해도 실행 됨(코드 읽기 힘듦)

```
public class Hello {

    public static void main(String[] args) {
        System.out.println("hello java");
    }
}
```
실행 결과
```
hello java
```
<hr>

```
public class hello
```
- hello를 **클래스**라고 함
- **파일명과 클래스 이름**이 같아야함
- {} 블록을 사용해서 클래스의 시작과 끝을 나타냄

```
public static void main(String[] args)
```
- main 메서드
- 자바는 main(String[] args) 메서드를 찾아서 프로그램 시작함
- {} 블록 사용해서 메서트의 시작과 끝을 나타냄
- **단축키 :** psvm -> enter 누르기

```
System.out.println("hello java");
```
- 값을 콘솔에 출력하는 기능
- 문자열 사용 시 " 큰따옴표 사용 "
- ;(세미콜론)으로 문장 구분
- **단축키 :** sout -> enter 누르기

## 07. 주석(comment)
**주석이란?** 코드를 이해하기 위한 설명을 적거나 특정 코드를 지우지 않고 실행을 막고 싶을 때 사용

**종류**
- 한 줄 주석
    - // 기호로 시작
- 여러 줄 주석
    - /*로 시작하고 */로 끝냄
## 08. 자바란?
표준 스펙과 구현으로 나뉨  
- **표준 스펙**
    - 자바는 이렇게 만들어야 한다는 설계도이자 문서
    - 표준 스펙을 기반으로 실제 작동하는 자바를 만듦
    - 자바 커뮤니티(JCP)를 통해 관리됨
- **구현**
    - 표준 스펙에 맞추어 실제 작동하는 자바 프로그램을 개발
    - 다양한 OS에서 작동하는 버전의 자바도 함께 제공함
    - 종류 : 오라클 Open JDK, Adoptium Eclipse Temurin, Amazon Corretto
만약 AWS를 쓴다면 Amazon Corretto 설치 추천

**JAVA 구현 장점**  
표준 스펙에 다 맞춰서 개발 되어 있기 때문에 다른 종류로 변경해도 대부분 문제 없이 작동함

**컴파일과 실행**  
- 자바 소스 코드를 개발자가 작성
- 자바 컴파일러를 사용해서 소스 코드를 컴파일 함
    - 자바가 제공하는 프로그램 사용
    - .java -> .class 파일 생성
    - 자바 소스 코드를 바이트코드로 변환하고 자바 가상 머신에서 더 빠르게 실행될 수 있게 최적화하고 문법 오류 검출
- 자바 프로그램 실행
    - 자바가 제공하는 java라는 프로그램 사용
    - 자바 가상 머신(JVM)이 실행되면서 프로그램이 작동
 
**IDE와 자바**  
- 컴파일
    - 자바 코드를 컴파일 하려면 javac 라는 프로그램을 직접 사용 -> 인텔리제이는 이 과정 자동 처리
    - 인텔리제이 화면에서 프로젝트에 있는 out 폴더 가면 컴파일된 .class 파일 확인 가능
- 실행
    - 자바를 실행하려면 java라는 프로그램을 사용 -> 컴파일된 .class 파일을 지정해주면 됨
    - 인텔리제이에서 자바 코드를 실행 시 컴파일과 실행을 모두 한번에 처리
 
**자바와 운영체제 독립성**  

일반적인 프로그램 특징
- 다른 운영체제에서 실행 불가능
- ex) 윈도우 프로그램 -> MAC이나 리눅스에서 작동 못함
- 이유 : 명령어가 호환되지 않음

자바 프로그램  
- 자바가 설치된 모든 OS에서 실행 가능
- 특정 OS에 맞추어 개발을 하지 않아도 됨 (OS 호환성 문제는 자바가 해결)
- 윈도우 자바는 윈도우 OS가 사용하는 명령어들로 구성되어 있음
- -> 개발자는 각 OS에 맞도록 자바를 설치하기만 하면 됨

**자바 개발과 운영 환경**  
- 개발할 때 자바와 서버에서 실행할 때 다른 자바를 사용할 수 있음
- 개발자들은 개발 편의를 위해 윈도우나 MAC OS를 주로 사용함
- 서버는 주로 리눅스 사용
- 자바의 운영체제 독립성 덕분에 각각의 환경에 맞추어 자바를 설치하는 것이 가능
