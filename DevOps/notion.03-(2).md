# Docker Compose
## 1. 정의 및 개념
### 1-1. 정의
**Docker Compose란?** 여러 개의 Docker 컨테이너를 한 번에 정의하고 실행하기 위한 도구   
(컨테이너 여러 개를 하나의 설정 파일로 묶어서 통째로 관리하는 시스템)

### 1-2. 개념
**YAML 파일** 하나로 여러 컨테이너의 설정을 정의하고 명령어 한 번으로 전체 시스템을 실행/중지하는 도구      
(여러 컨테이너를 하나의 애플리케이션처럼 묶어서 관리)

## 2. 구조 
### 2-1. Services
**Services란?** 실행할 컨테이너들을 정의하는 영역    
- 하나의 서비스 = 하나의 컨테이너
- 여러 개 서비스 가능
- Docker Compose의 핵심 영역

**주요 옵션**   
| 항목 | 설명 |
| ----- | ----- |
| image | 사용할 이미지 |
| build | Dockerfile로 직접 빌드 |
| ports | 외부와 연결 |
| environment | 환경 변수 |
| depends_on | 실행 순서 |
| networks | 연결 네트워크 |

### 2-2. Networks
**Networks란?** 컨테이너끼리 서로 통신하기 위한 연결 구조
- 같은 네트워크에 있어야 통신 가능
- 컨테이너끼리 IP대신 서비스 이름으로 통신
- 내부 전용 네트워크 생성 가능

### 2-3. volumes
**Volumes란?** 컨테이너의 데이터를 외부 저장소에 저장하는 구조  
- 컨테이너는 휘발성 -> 데이터 날아감
- Volumes로 데이터 영구 저장 가능
- DB에서 거의 필수

## 3. YAML
**YAML(YAML Ain’t Markup Language)란?** 데이터를 사람이 읽기 쉽게 표현하는 설정 파일 형식

### 3-1. 핵심 기본 구조
**Key - Value**     
형태 - 이름 : 값    
데이터를 단순하게 표현할 때 사용함

**계층 구조**   
들여쓰기를 이용해서 데이터 안에 데이터가 포함되는 구조를 표현   
(상위 키 안에 하위 키가 존재하는 형태)

**리스트(배열)**     
같은 종류의 여러 데이터를 나열할 때 사용하는 구조   
(각 항목 앞에 하이픈(-)을 붙여 표현함)  

### 3-2. 특징
**사람이 읽기 쉬움**    
구조가 단순해서 사람이 보기 편하게 만들어짐
- {},[] 같은 복잡한 기호 없음
- 문법이 거의 영어 문장 형태처럼 보임
- 불필요한 장식 제거

**들여쓰기로 구조 표현**    
들여쓰기 자체가 구조의 의미를 가짐
- 공백 개수 = 계층 구조
- 탭 사용 안함

**데이터 중심 형식**    
실행용 코드가 아닌 데이터 표현용
- 실행 코드 작성 X
- 값과 구조만 정의
- 프로그램이 읽어서 실행


## 3. 특징
**멀티 컨테이너 통합 관리**     
여러 개 컨테이너를 하나의 애플리케이션처럼 묶어서 관리하는 구조
- web / db / redis 등을 하나의 yml로 정의
- docker-compose up 한 번으로 전체 실행

**선언형 설정 방식**    
어떻게 실행할지가 아닌 무엇을 실행할지만 정의
- 명령어로 하나씩 실행 하는 것이 아닌
-> YAML에 상태를 정의

**환경 재현성 및 자동 네트워크 연결**     
환경 재현성
- 같은 docker-compose면 어디서 실행해도 동일한 환경
- 개발/서버/팀원 환경 차이 제거

자동 네트워크
- Compose 실행 시 자동으로 내부 네트워크 생성
- 컨테이너끼리 IP 없이 서비스 이름으로 통신

**서비스 실행 순서 제어**   
컨테이너 실행 순서를 정의할 수 있음
- DB를 먼저 실행하고 Web 실행함
- DB 준비 완료까지는 보장 안 함

## 4. 장점
**환경 설정이 코드처럼 관리됨**     
모든 설정을 YAML파일 하나에 저장
- Git으로 버전 관리 쉬움
- 서버 설정 = 코드 관리

**환경 재현성**     
같은 Compose 파일이면 어디서 실행해도 동일 환경

**네트워크 자동 구성**  
컨테이너끼리 자동으로 연결됨
- 자동 네트워크 생성
- 서비스 이름으로 접근 가능

## 5. 단점
**대규모 시스템 관리 한계**     
- 자동 확장 없음
- 분산 시스템 기능 부족

**자동 복구/장애 대응 기능 부족**   
컨테이너가 죽으면 자동 복구 기능 약함
- 단순 실행 도구
- 고급 운영 기능 없음

**자동 확장 기능 제한적**  
트래픽 증가 시 자동 확장 부족  
- 컨테이너 수 자동 증가 X
- 부하 분산 자동 처리 X
