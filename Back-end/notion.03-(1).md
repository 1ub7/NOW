# 웹 애플리케이션 계층 구조
## 1. 개념
**웹 애플리케이션 계층 구조란?** 웹 프로그램의 기능을 역할별로 나누어 각각의 계층이 독립적으로 동작하도록 구성한 소프트웨어 구조임      
(코드의 역할을 분리해서 유지보수성과 확장성을 높이기 위한 구조)

**왜 나눔?**    
- 각 계층이 자기 역할만 하도록 분리하기 위해
- 코드 재사용 가능해서
- 한 부분만 수정하면 되도록 만들기 위해

## 2. 전체 구조
```
Client(사용자)
   ↓
Controller
   ↓
Service
   ↓
Repository
   ↓
Database(DB)
   ↑
Entity / DTO
```

## 3. 각 계층 역할
**Client**  
요청을 보내는 주체 
- HTTP 요청을 서버로 보냄
- 결과(응답)를 화면에 출력 
- ex) 브라우저, 앱, 프론트엔드 

**Controllor**  
사용자의 요청을 받아서 Service로 전달하고 응답을 반환하는 계층
- HTTP 요청 처리
- URL 매핑
- Service 호출
- 응답 반환     

특징
- 비즈니스 로직을 가지지 않음
- 단순 전달 역할만 수행

**Service**     
애플리케이션의 핵심 비즈니스 로직을 처리하는 계층   
- 데이터 가공
- 여러 Repository 조합
- 규칙/조건 처리
- 트랜잭션 처리

특징
- 가장 중요한 로직 계층
- 재사용 가능

**Repository**  
데이터베이스와 직접 연결되는 계층   
- DB의 CRUD 작업 처리
- SQL 실행 또는 ORM 처리
- 데이터 영속성 담당

특징    
- DB 접근만 담당
- 비즈니스 로직 없음

**Entity**      
데이터베이스 테이블과 1:1로 매핑되는 객체
- DB 구조 표현
- 실제 데이터 저장 구조
- ORM에서 테이블 역할

특징
- DB 테이블 그대로 표현
- 실제 데이터 구조

**DTO(Data Transfer Object)**   
계층 간 데이터 전달용 객체  
- 필요한 데이터만 전달
- Controller <-> Service 사이 이동
- 보안/최적화 목적

특징    
- Entity와 분리됨
- 데이터 전달 전용

## 4. 특징
**역할 기반 분리 구조**     
각 계층이 자기 역할만 수행하도록 분리됨     
(하나의 기능을 한 곳에서 다 처리하지 않고 역할별로 쪼개서 책임을 나눔)
- 코드가 섞이지 않음
- 수정 범위 줄어듦

**유지보수성 향상**     
한 계층만 수정해도 전체 영향이 줄어듦   
(한 곳 수정 = 전체 시스템 안정성 유지)
- 변경 영향 범위 제한
- 버그 발생 범위 축소

**재사용성 증가**   
Service 같은 핵심 로직은 여러 Controller에서 사용 가능
- 비즈니스 로직이 한 곳에 집중됨
- (Controller는 단순 전달 역할)

## 5. 장점
**유지보수성 향상**     
코드 수정이 특정 계층에만 영향을 주도록 설계된 구조     
(코드 수정이 부분 수정으로 끝남)
- Controller, Service, Repository가 분리되어 있음
- 수정 범위가 작아짐

**재사용성 증가**   
같은 로직을 여러 곳에서 반복해서 쓰지 않아도 되는 구조

왜 가능함?
- 핵심 로직이 Service에 집중됨
- Controller는 단순 호출 역할

**테스트 용이성 증가**  
각 계층을 독립적으로 테스트 가능

왜 좋음?
- Service만 따로 테스트 가능
- DB 없이 로직 테스트 가능
- Controller도 단독 테스트 가능

## 6. 단점
**구조가 복잡해짐**     
단순 기능도 여러 계층을 거쳐야 함   

문제점
- 단순 기능인데도 구조가 길어짐
- 코드 이해할 때 여러 파일 봐야 함
- 흐름이 직관적이지 않음

**코드량 증가**     
같은 기능을 만들기 위해 작성해야 할 코드 파일이 많아지는 현상
- DTO 생성 필요
- Entity 분리 필요
- Service/Repository 분리 필요

**흐름 추적이 어려울 수 있음**  
요청이 여러 계층을 거치기 때문에 문제가 발생했을 때 원인 찾기가 어려운 문제

왜 어려워짐?
- 코드가 여러 파일에 분산됨
- 디버깅 시 이동이 많음
- 데이터 변환 과정 많음
