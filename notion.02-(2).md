# HTTP / HTTPS
## 1. HTTP와 HHTPS에 대해
### 1-1. HTTP
(**H**yper**T**ext **T**ransfer **P**rotocol)   
**HTTP란?** 웹 브라우저와 웹 서버가 데이터를 주고받는 규칙

### 1-2. HTTPS
(**H**yper**T**ext **T**ransfer **P**rotocol **S**ecure)
**HTTPS란?** 기본 HTTP에 SSL/TLS 보안 계층이 추가된 것

## 2. HTTP와 HTTPS 공통점
**웹 통신 프로토콜**  
클라이언트 <-> 서버 간의 통신 규칙   
- 인터넷 상에서 웹 서비스를 가능하게 하는 기본 규칙
- 클라이언트가 요청 -> 서버가 그 요청을 처리해서 응답을 반환

**요청-응답 방식 사용**  
비동기적 요청-응답 구조 사용  
- 클라이언트가 먼저 요청을 보내야 서버가 응답
- 서버는 요청 없이 데이터 안 보냄
- 상태를 유지하지 않는 Stateless 구조
  
**URL 사용**  
웹 주소인 URL을 통해 자원 접근     
- URL은 "어디에 있는 어떤 자원인지"를 지정
- HTTP/HTTPS는 이 URL을 해석해서 서버와 연결

**동일한 HTTP 메서드 사용**  
내부적으로 동일한 메서드 사용  
(4번 참고)

## 3. HTTP와 HTTPS 차이점
| 항목 | HTTP | HTTPS |
| ----- | ----- | ------ |
| 암호화 | X | O |
| 보안성 | 낮음 | 높음 |
| 기본 포트 | 80 | 443 |
| 인증서 | X | O |
| 주소 표시 | http:// | https:// |
| 속도 | 약간 빠름 | 약간 느릴 수 있음 |
| 신뢰성 | 낮음 | 높음|
| 사용 환경 | 테스트 | 실제 서비스 |

## 4. HTTP와 HTTPS의 메서드
**메서드(Method)란?** 클라이언트가 서버에게 무엇이 하고 싶은지 알려주는 요청 방식   

### 4-1. 주요 메서드
**GET** : 서버에서 데이터를 가져옴
- 캐싱 및 북마크 가능
- 서버 데이터 변경 안 함
- 주소창에 표시 가능

**POST** : 서버에 데이터를 보냄
- 데용량 데이터 전송 가능
- URL에 데이터 노출 안 됨
- 같은 요청 반복 시 중복 생성 가능

**PUT** : 기존 데이터를 전체 수정
- 멱등성 보장
- 일부 수정용보단 전체 교체용
- 기존 자원이 없으면 새로 생성 가능
- 같은 요청 반복해도 결과 동일

**PATCH** : 일부 데이터만 수정
- 필요한 부분만 변경
- 네트워크 효율적
- 전체 데이터 다시 안 보내도 됨

**DELETE** : 데이터 삭제
- 서버에서 제거 요청
- 멱등성 있음
- 실제 삭제 대신 상태만 변경할 수도 있음

**HEAD** : 헤더 정보만 요청 / 본문 안 받음
- 응답 정보만 확인 가능
- 본문 없이 속도 빠름
- 파일 존재 여부 확인 가능
- 다운로드 전 크기 확인 가능

**OPTIONS** : 서버가 지원하는 메서드 확인
- 통신 전 확인용
- CORS 처리 사용
- 서버가 허용하는 메서드 확인
- API 개발에서 중요

위에서 나온 **멱등성(Idempotent)이란?**     
같은 요청을 여러 번 보내도 결과가 같다는 것

### 4-2. 주요 메서드 정리

| 메서드 | 의미 | 사용 목적 |
| ----- | ----- | ----- |
| GET | 조회 | 데이터 읽기 |
| POST | 생성 | 데이터 전송 |
| PUT | 전체 수정 | 전체 변경 |
| PATCH | 부분 수정 | 일부 변경 |
| DELETE | 삭제 | 데이터 삭제 |
| HEAD | 헤더 확인 | 정보 조회 |
| OPTIONS | 지원 확인 | 기능 확인 |
