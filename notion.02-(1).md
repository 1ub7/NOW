# 객체지향
## 1. 객체지향이 무엇인가?
(**O**bject-**O**riented **P**rogramming)   

**객체지향이란?** 데이터와 기능을 하나로 묶은 객체 중심으로 프로그램을 만드는 방식  
즉, 현실 세계의 사물을 프로그램 안의 객체로 표현하는 방법

**기본 구조?**   
클래스 -> 객체 -> 메서드 -> 동작

### 1-1. 클래스(Class)
**클래스란?** 객체를 만들기 위한 설계도 
객체가 가져야 할 데이터와 기능을 미리 정의  
(즉, 객체를 만들기 위한 틀)

### 1-2. 객체(Object)
**객체란?** 객체는 클래스를 기반으로 만들어진 실제 존재하는 데이터  
(즉, 클래스의 실제 결과물)

### 1-3. 속성(Attribute)
**속성이란?** 객체가 가지고 있는 정보나 상태값  
-> 객체마다 다른 값을 가질 수 있음  
(즉, 객체의 정보)

### 1-4. 메서드(Method)
**메서드란?** 객체가 수행할 수 있는 동작이나 기능
(즉, 객체의 행동)

### 1-5. 생성자(Constructor)
**생성자란?** 객체가 생성될 때 자동으로 실행되는 특별한 메서드  
-> 객체를 만들 때 초기값 설정할 때 사용     
(즉, 객체 생성 초기 설정)

### 1-6. 인스턴스(Instance)
**인스턴스란?** 특정 클래스로부터 생성되어 소속된 객체를 의미

## 2. 객체지향의 핵심 개념
### 2-1. 캡슐화(Encapsulation)
**캡슐화란?** 데이터와 기능을 하나로 묶고 외부에서 직접 접근하지 못하게 보호하는 것   
**왜 사용할까?** 객체 내부 데이터를 안전하게 관리하기 위해 사용

### 2-2. 상속(Inheritance)
**상속이란?** 기존 클래스의 속성과 기능을 새로운 클래스가 물려받는 것   
**왜 사용할까?** 코드 재사용이 가능

### 2-3. 다형성(Polymorphism)
**다형성이란?** 같은 이름의 기능이 객체마다 다르게 동작하는 것  
**왜 사용할까?** 같은 명령으로 다양한 객체 처리 가능

### 2-4. 추상화(Abstraction)
**추상화란?** 복잡한 내부는 숨기고 필요한 기능만 보여주는 것    
**왜 사용할까?** 사용하기 쉽게 만들기 위해

## 3. 객체지향의 장점
### 3-1. 코드 재사용성이 높음
상속을 통해 이미 만들어 놓는 클래스를 다시 사용할 수 있음   
(비슷한 기능을 가진 클래스를 새로 만들 때 처음부터 작성하지 않아도 됨)

**효과**    
_ 코드 중복 감소    
_ 개발 시간 단축    
_ 오류 감소 

### 3-2. 유지보수가 쉬움
객체별로 나누어져 있어서 문제가 생겼을 때 수정 위치를 쉽게 찾을 수 있음     
(관련 코드가 한 클래스 안에 모여 있음)

**효과**    
_ 오류 수정 쉬움    
_ 기능 추가 쉬움    
_ 코드 관리 편리

### 3-3. 현실 세계 표현이 자연스러움
객체지향은 현실의 사물을 그대로 코드로 표현하기 좋음  
(즉, 생각한 구조 그대로 코드로 옮기기 쉬움)  
-> 사람이 프로그램 구조를 이해하기 쉬워짐   

### 3-4. 협업이 편함
객체별로 역할 분리  
-> 각자 자기 객체만 개발하면 되기 때문에 서로의 간섭이 줄어듦

**효과**    
_ 충돌 감소     
_ 작업 분담 쉬움    
_ 대규모 개발 유리

### 3-5. 보안성 높음
캡슐화를 통해 중요한 데이터를 외부에서 직접 변경하지 못하게 만들 수 있음    
(메서드를 통해서만 변경 가능)   
-> 데이터 보호에 유리   

### 3-6. 확장이 쉬움
기존 코드를 크게 수정하지 않고 새 클래스를 추가     
-> 큰 프로그램에 유리

## 4. 객체지향의 단점
### 4-1. 설계 복잡
코드를 바로 작성하는 것이 아니라 먼저 구조를 생각해야함     
(초반 설계 어려움)

**생각해야 할 것**  
_ 어떤 객체가 필요한가  
_ 어떤 속성이 필요한가  
_ 어떤 기능이 필요한가  
_ 객체 관계는 무엇인가

### 4-2. 작은 프로그램 비효율적
간단한 프로그램인데도 클래스, 객체, 메서드를 만들어야 해서 코드가 길어짐    
-> 객체지향으로 하면 더 복잡해질 수 있음

### 4-3. 실행속도가 느릴 수 있음
객체 생성과 메서드 호출 과정이 추가되므로 절차지향보다 느려질 수 있음   
-> 성능이 중요한 경우 단점이 될 수 있음     

**객체지향 내부과정**    
_ 메모리 할당   
_ 참조 관리     
_ 메서드 호출

### 4-4. 메모리 사용량 많음
객체 하나마다 데이터와 메서드 정보가 저장되기 때문  

**객체 수가 많아지면 일어나는 일**  
_ 메모리 사용 증가  
_ 성능 저하
